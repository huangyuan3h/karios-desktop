# Market Sentiment 模块

> 市场情绪分析与仓位建议模块

---

## 模块定位

Market Sentiment 模块通过分析 A 股市场的多项指标，计算市场情绪状态，提供"红绿灯"式的风险提示和仓位建议。它是**仓位管理**的核心参考工具。

## 核心价值

- **市场温度计**：直观展示市场强弱状态
- **仓位建议**：根据市场状态给出建议仓位比例
- **风险预警**：提前识别市场过热或过冷信号
- **历史对比**：查看近期市场情绪变化趋势

---

## 核心概念

### 指数信号（红绿灯）

系统跟踪上证指数和创业板指，给出红绿黄信号：

| 信号 | 条件 | 建议仓位 | 含义 |
|------|------|----------|------|
| 🔴 Red | Price < MA20 或 MA5 < MA20 | 0%-10% | 趋势走坏，防守为主 |
| 🟡 Yellow | Price > MA20 但条件不完美 | 30% | 震荡市，谨慎操作 |
| 🟢 Green | Price > MA20 且 MA5 > MA20 且 MA20↑ 且量比>1 | 50%-60% | 趋势向上，可积极参与 |
| ❇️ Deep Green | MA5 > MA20 > MA60 且 Price > MA5 且 Breadth > 60% 且量比>1.2 | 80%-100% | 强势市场，可重仓 |

### 市场状态判断

系统会综合两个指数的信号，给出整体市场判断：

| 状态 | 条件 | 操作建议 |
|------|------|----------|
| **Strong（强势）** | 上证和创业板都为 Green | 顺势为主，控制仓位与回撤 |
| **Diverging（分化）** | 一绿一红或黄 | 严禁追高，仅限防守型回踩，买入仅用反弹买入策略单 |
| **Weak（弱势）** | 都不是 Green | 防守为主，严格控制风险，买入仅用反弹买入策略单 |

### 风险模式

| 模式 | 说明 | 建议 |
|------|------|------|
| hot | 市场火热，情绪高涨 | 注意风险，不要追高 |
| euphoric | 市场极度亢奋 | 逐步减仓，锁定利润 |
| caution | 市场谨慎，风险存在 | 控制仓位，避免重仓 |
| no_new_positions | 不建议新开仓 | 保持观望，只做止损 |

### 关键指标

#### 涨跌比（Up/Down Ratio）

```
涨跌比 = 上涨家数 / 下跌家数

> 1：多头占优
< 1：空头占优
```

#### 昨日涨停溢价（Yesterday Limit-Up Premium）

```
溢价 = 昨日涨停股今日平均涨幅

正值：涨停股第二天继续涨，市场接力意愿强
负值：涨停股第二天补跌，市场接力意愿弱
```

#### 涨停失败率（Failed Limit-Up Rate）

```
失败率 = 今日涨停后打开的股票数量 / 今日触及涨停的股票总数

高失败率：打板风险大，资金分歧严重
低失败率：封板成功率高，资金共识强
```

#### 市场成交额（Market Turnover）

```
成交额 = 两市总成交金额

放量：资金活跃
缩量：资金观望
```

## 业务流程

### 日常使用流程

```
1. 盘后同步
   ├── 点击 "Sync sentiment" 更新当日数据
   └── 或在 Dashboard 点击 "Sync all"

2. 查看信号
   ├── 查看上证指数和创业板指信号
   ├── 查看整体市场状态
   └── 查看建议仓位

3. 分析指标
   ├── 查看涨跌比
   ├── 查看涨停溢价
   ├── 查看涨停失败率
   └── 查看成交额

4. 历史对比
   ├── 查看最近 5 天的数据
   └── 观察趋势变化

5. 决策参考
   ├── 根据仓位建议调整仓位
   └── 结合其他模块做决策
```

## 与其他模块的关系

```
                    ┌──────────────────┐
                    │ Market Sentiment │
                    └────────┬─────────┘
                             │
        ┌────────────────────┼────────────────────┐
        │                    │                    │
        ▼                    ▼                    ▼
 ┌────────────┐       ┌────────────┐       ┌────────────┐
 │  Watchlist │       │  Industry  │       │    AI      │
 │ (买入规则) │       │   Flow     │       │ (市场分析) │
 └────────────┘       └────────────┘       └────────────┘
```

### 与 Watchlist 的关系

- Watchlist 的"强制买入规则"依赖市场信号
- 只有在 Green Zone（双绿）时，A_pullback/wait 才可能被强制改为 buy
- 非绿环境下，买入需更加谨慎

### 与 Industry Flow 的关系

- 市场情绪影响板块选择
- 弱势市场下，只有最强的主线板块值得关注
- 强势市场下，可以适当放宽板块选择

### 与 AI 的关系

- 市场情绪数据可作为 AI 分析上下文
- AI 可以解释市场状态的原因
- AI 可以结合新闻解释市场走势

## 典型使用场景

### 场景一：每日仓位决策

1. 盘后查看市场信号
2. 根据红绿灯确定当前仓位范围
3. 结合个人风险偏好调整
4. 规划明天的操作策略

### 场景二：风险预警

1. 关注涨停失败率
2. 关注涨停溢价
3. 失败率升高 + 溢价转负 = 风险信号
4. 考虑减仓或观望

### 场景三：情绪拐点识别

1. 对比多日数据
2. 识别情绪拐点
3. 情绪从低到高：可能是入场机会
4. 情绪从高到低：可能是离场信号

## 关键业务规则

### 仓位建议与风险承受能力

| 市场状态 | 保守型 | 稳健型 | 激进型 |
|----------|--------|--------|--------|
| Deep Green | 60% | 80% | 100% |
| Green | 40% | 50% | 60% |
| Yellow | 20% | 30% | 40% |
| Red | 0% | 5% | 10% |

### 买入策略选择

| 市场状态 | 建议策略 |
|----------|----------|
| Deep Green | 可用 A_pullback 和 B_momentum |
| Green | 主要用 A_pullback |
| Yellow | 只用反弹买入策略单 |
| Red | 不建议买入 |

### 止损规则

无论市场状态如何，都应严格执行止损：

- 单只股票亏损超过 10%：考虑止损
- TrendOK 变为 ❌：考虑止损
- 止损信号触发：立刻离场

## 数据来源

- 指数数据：上证指数、创业板指
- 市场数据：涨跌家数、成交额
- 涨停数据：涨停股数量、失败率、溢价

数据在盘后更新，盘中数据仅供参考。

---

## 注意事项

### 滞后性

- 市场情绪指标有一定滞后性
- 不适合作为唯一决策依据
- 应结合其他分析工具

### 极端情况

- 极端行情下，指标可能失真
- 例如：系统性风险下，所有信号都可能失效
- 需要人工判断

### 持续监控

- 市场状态可能在短期内变化
- 建议每日查看
- 重大事件后及时更新
